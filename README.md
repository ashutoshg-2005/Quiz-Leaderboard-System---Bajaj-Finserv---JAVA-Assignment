# JAVA & Salesforce - Question Paper - Bajaj Finserv Health Limited

## Quiz Leaderboard System

This is a simple Java console application made for the Quiz Leaderboard System assignment.

The main goal of this program is to:

1. Call the validator API 10 times.
2. Collect all quiz score events.
3. Ignore duplicate records.
4. Calculate total score for each participant.
5. Create a leaderboard.
6. Submit the final leaderboard to the API.

## Code Flow

The program starts from the `main()` method.

Basic flow:

```text
main()
  -> read registration number
  -> poll API from 0 to 9
  -> parse events from response
  -> skip duplicate events
  -> add scores participant-wise
  -> sort leaderboard
  -> build submit JSON
  -> submit final answer
```

## Functions Used in the Code

### `main(String[] args)`

This is the starting point of the program.

It does most of the main work:

1. Reads registration number from command line.
2. Checks if dry run mode is enabled.
3. Creates `HashMap` for participant scores.
4. Creates `HashSet` for duplicate checking.
5. Runs poll loop from `0` to `9`.
6. Prints final leaderboard.
7. Submits the final answer if dry run is not enabled.

### `getMessages(String regNo, int poll)`

This function calls the GET API.

It creates this URL:

```text
https://devapigw.vidalhealthtpa.com/srm-quiz-task/quiz/messages?regNo=REG_NO&poll=POLL_NUMBER
```

It returns the API response as a `String`.

This function is separate so the API calling code does not mix too much with the score calculation logic.

### `submitLeaderboard(String payload)`

This function calls the POST API:

```text
/quiz/submit
```

It sends the final leaderboard JSON to the server.

The `payload` is the JSON created by `buildSubmitPayload()`.

### `makeLeaderboard(Map<String, Integer> participantScores)`

This function creates the final leaderboard list.

The score data is first stored in a `HashMap` like this:

```text
Alice -> 100
Bob   -> 120
```

But for submission, we need a leaderboard list.

So this function converts the map into a list of `LeaderboardRow` objects and sorts it by highest score.

### `parseEvents(String json)`

This function reads the API response and extracts all quiz events from the `"events"` array.

Each event has:

```text
roundId
participant
score
```

This function returns a `List<QuizEvent>`.

### `getTextValue(String json, String key)`

This function extracts text values from a small JSON object.

Example:

```json
{ "participant": "Alice" }
```

If the key is `participant`, it returns:

```text
Alice
```

It is used for `roundId` and `participant`.

### `getNumberValue(String json, String key)`

This function extracts number values from JSON.

Example:

```json
{ "score": 10 }
```

If the key is `score`, it returns:

```text
10
```

It is used for reading the score.

### `buildSubmitPayload(String regNo, List<LeaderboardRow> leaderboard)`

This function builds the final JSON body for submission.

Example output:

```json
{
  "regNo": "RA2311033010184",
  "leaderboard": [
    {
      "participant": "Alice",
      "totalScore": 100
    }
  ]
}
```

The actual JSON generated by the program is in one line, which is valid for API submission.

### `encode(String value)`

This function makes the registration number safe to use inside a URL.

It uses:

```java
URLEncoder.encode(...)
```

This is useful if the registration number contains any special character.

### `cleanJsonText(String value)`

This function makes text safe before adding it into JSON.

For example, if text contains double quotes, it escapes them so the JSON does not break.

### `QuizEvent` Class

This is a simple class used to store one event from the API.

It has:

```java
String roundId;
String participant;
int score;
```

Example:

```text
R1, Alice, 10
```

### `LeaderboardRow` Class

This class stores one row of the final leaderboard.

It has:

```java
String participant;
int totalScore;
```

Example:

```text
Alice, 100
```

## Duplicate Handling Logic

This is the most important part of the program:

```java
String uniqueKey = event.roundId + "|" + event.participant;
```

Then the program checks:

```java
if (alreadyCounted.contains(uniqueKey)) {
    duplicateCount++;
    continue;
}
```

Meaning:

1. If the same round and participant already exists, skip it.
2. If it is new, add it to the `HashSet`.
3. Then add the score to the participant total.

Example:

```text
Poll 1: R1 Alice 10   counted
Poll 3: R1 Alice 10   skipped as duplicate
```

So Alice gets only 10, not 20.

## Why I Used HashMap and HashSet

### HashMap

I used `HashMap` because I need to store each participant's total score.

Example:

```java
participantScores.put(event.participant, oldScore + event.score);
```

This updates the participant's total score.

### HashSet

I used `HashSet` because I need to check duplicates quickly.

Example:

```java
alreadyCounted.contains(uniqueKey)
```

If the key already exists, the event is duplicate.

## Sorting Logic

The leaderboard is sorted by total score in descending order.

That means highest score comes first.

If two participants have the same score, their names are sorted alphabetically.
